لما الكود بيجيب معلومات المستخدم من Salla API, الاستجابة بتيجي بالشكل ده:

```
استجابة Salla API الفعلية:
┌─────────────────────────────────────────┐
│ {                                       │
│   "data": {                             │
│     "email": "sala.helpcenter@gmail.com" │  ← الإيميل هنا! (تحت data)

│   },                                    │

│   "success": true                       │
│ }                                       │

```

لكن الكود بيدوّر على الإيميل في المكان الغلط:

```
الكود بيدوّر هنا:          الإيميل موجود هنا فعلاً:
──────────────────         ──────────────────────────
u.email          ❌ مش موجود    u.data.email      ✅ موجود

u.user.email     ❌ مش موجود    u.data.user       ✅ ممكن
```

## المشكلة الثانوية: `return` بيخرج من الكود بدري

في ملف `webhook.ts` سطر 175، لو الدومين مش موجود في الـ payload والكود جابه من API:

```
الكود بيعمل:
  1. يجلب الدومين من API   ✅
  2. يحفظ الدومين           ✅
  3. return;                ← يخرج من كل الكود!
  4. إرسال الإيميل          ← ما يوصلوش أصلاً
```

**ملاحظة:** بناءً على التوقيتات، المشكلة دي **على الأرجح ما حصلتش** في هذا المتجر بالذات — لأن الدومين و userinfo اتحفظوا في نفس اللحظة. لكنها **مشكلة حقيقية** ممكن تأثر على متاجر تانية في المستقبل.

---

## مين المتأثر؟

```
  المشكلة الأساسية (بنية الاستجابة):
  ────────────────────────────────────
    تأثر على: كل المتاجر ❌
    لأن: infoEmail دائماً فاضي بسبب الـ data wrapper
    لكن: لو fetchStoreInfo نجح ورجع إيميل → الإيميل بيتبعت
    يعني: المتاجر اللي fetchStoreInfo بتنجح فيها → الإيميل بيوصل
           المتاجر اللي fetchStoreInfo بتفشل فيها → الإيميل ما بيوصلش

  المشكلة الثانوية (return):
  ──────────────────────────
    تأثر على: المتاجر بدومين مخصص فقط
    لأن: المتاجر بدومين salla.sa الدومين بيكون في الـ payload
         فالكود ما بيدخلش في البلوك اللي فيه return
```

## الإصلاح المطلوب

### إصلاح 1 — الأهم: فك طبقة `data` قبل استخراج الإيميل

**الملف:** `src/pages/api/salla/webhook.ts` — سطر 350

```
قبل:
  const u = uinfo as Dict;
  const infoEmail = u.email ?? u.merchant.email ?? u.user.email;
  // النتيجة: دائماً undefined

بعد:
  const u = uinfo as Dict;
  const uData = (u.data && typeof u.data === "object") ? u.data : u;
  const infoEmail = uData.email ?? uData.merchant.email ?? uData.user.email;
  // النتيجة: "sala.helpcenter@gmail.com" ✅
```

### إصلاح 2: حذف `return` من سطر 175

```
قبل:
  return; // Exit early after successful fetch

بعد:
  // تم حفظ الدومين — نكمل المعالجة
```

### إصلاح 3: تغيير مستوى اللوج من `debug` لـ `warn`

سطر 431 — لما `targetEmail` يكون فاضي، اللوج مستواه `debug` وده بيخليه مش واضح. يتغير لـ `warn`.

## ملخص المشاكل

| المشكلة | الخطورة | التأثير | الإصلاح |
|---------|---------|---------|---------|
| بنية استجابة userinfo مش متوافقة | عالية جداً | `infoEmail` دائماً فاضي لكل المتاجر | فك طبقة `data` قبل استخراج الحقول |
| `return` في بلوك الدومين | متوسطة | يخرج من الكود لمتاجر الدومين المخصص | حذف `return` من سطر 175 |
| لوج مستوى `debug` عند عدم وجود إيميل | منخفضة | صعب تتبع المشكلة | تغيير لمستوى `warn` |
